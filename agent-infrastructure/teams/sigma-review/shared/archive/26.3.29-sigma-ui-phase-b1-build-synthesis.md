### 4. DA Challenges That Changed the Build

**DA[#1] CRITICAL (plan phase) — FIX-1 mechanism mismatch:**
Challenge: ADR[2]'s "subsection-header-match" mechanism for §2a-§2e does not exist. `tech-architect.md:77-105` shows `## Analytical Hygiene` is ONE section; §2a-§2e are inline bullet items, not separate headers. `_match_rewrite_key()` operates on headers only. The described mechanism could never fire. CQA did not catch this (did not inspect agent .md files). IE-2 independently confirmed same finding.
Result: ADR[2] mechanism revised from subsection-header-match → whole-section-rewrite. This was the only critical plan-blocking finding. P(plan-ready) recovered from 0.72 → 0.91 after resolution.

**DA[#2] HIGH (plan phase) — IC5 gate lifecycle = dead code:**
Challenge: fire_gate/set_active_gate/resolve_gate have zero B1 consumers. `active_gate=None` hardcoded. Gate UX is a Phase B2 Streamlit UI feature. TA partial-defend: type definitions kept (cheap, B2 needs them); lifecycle methods deferred.
Result: IC[B1-4] scope reduced to types-only. SQ[IE-4] reduced. ~4-6 lifecycle methods not written.

**DA[#3] MEDIUM (plan phase) — dual validation authority:**
Challenge: ADR[5] creating both StrEnum AND independent frozenset = maintenance drift risk (add to enum, forget frozenset, or vice versa).
Result: frozenset derived from enum. Single canonical source. ADR[5] revised.

**DA[#4] HIGH (plan phase) — GAP-2 write-order timing:**
Challenge: IC[B1-3] says "on transition" without specifying before/after the `self.current_round = 1` reset in `record_phase_transition()`. If registry write happens AFTER reset, all historical phases show completed_rounds=1 regardless of actual count.
Result: IC[B1-3] revised to explicit write-order constraint. CQA-5 and IE-4 independently confirmed same issue (triple convergence). Constraint documented in code comment at review_state.py:201-202.

**DA[#5] PROCESS (plan phase) — §2d source tagging 0/17:**
Challenge: 17 plan items had zero `|source:{type}` tags in violation of §2d build-directives.
Result: All 17 items tagged retrospectively. Tags verified plausible against code inspection evidence.

**DA[#6] MEDIUM (plan phase) — FIX-2 cascade underspecified:**
Challenge: IC[B1-2] listed 3 methods in the parameter chain; actual chain is 5 methods (omitted `_load_agent_prompt` and `_strip_sections`). CQA-6 independently confirmed same count.
Result: IC[B1-2] revised to enumerate full 5-method chain explicitly.

**DA[#7] PROCESS (plan phase) — OQ[2] DS2 ReviewMode gold-plating:**
Challenge: DS2 ReviewMode not in 7 confirmed work items. "For completeness" = §4c gold-plating signal.
Result: OQ[2] struck from scope. No ReviewMode enum added to B1.
