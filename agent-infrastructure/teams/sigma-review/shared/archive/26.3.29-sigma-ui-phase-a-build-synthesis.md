### 3. Key Architecture Decisions (from plan phase)

**ADR[1]: importlib.util.spec_from_file_location for orchestrator import** (not sys.path mutation)
— Modules cached on wrapper instance, not globally. cmd_* functions NEVER called — only `build_analyze_workflow()`/`build_build_workflow()` and direct gate_checks calls. Permanent constraint. `OrchestratorWrapper.get_state()` returns cached `OrchestratorState` from last `start()`/`advance()` return value — no `_make_state()` private attr access on normal path. Exception: `restore_checkpoint()` calls `_make_state()` with documented justification (state reconstruction after restore, no public API alternative).

**ADR[2]: REWRITE approach for agent .md processing** (not KEEP_SECTIONS)
— KEEP verbatim: Role, Expertise, Weight, Dialectical Bootstrapping. REWRITE: §2h (Cross-Model Verification) and §2a-§2e (Analytical Hygiene) — replace tool-call instructions with SDK-context equivalents. EXCLUDE: Boot, Comms, Persistence, Promotion, Research, Convergence, Timeout. Mode filter: strip BUILD-Mode sections in ANALYZE dispatch; strip ANALYZE hints in BUILD dispatch. SDK unavailability block mandatory in every system= prompt.

**ADR[3]: 7-item context injection** (memory, peer_plans, decisions, task_description, sigma_comm, comms_format, workspace_target) + MEMORY ENTRY extraction with 200-char fallback
— sigma-comm.md load-bearing: without it SDK-dispatched agents produce plain prose that gate_checks.py regex patterns do not recognize. persist_post_response() non-fatal: logs ContextError and continues, never discards AgentResult.

**ADR[4]: TIER-A observables additive to gate_checks.py**
— `extract_tier_a_signals()` and `check_tier_a_coverage()` in `tier_a_observables.py`. New `compute_analyze_belief_with_tier_a()` wrapper — `compute_analyze_belief()` in gate_checks.py NOT modified. tier_a_weight normalized so zero TIER-A signals produce zero contribution. Regression anchor test (`test_gate_checks_tier_a_regression.py`) captured baseline posteriors before edit; posteriors unchanged when TIER-A signals are absent.

**ADR[5]: Thin async adapter at method boundary**
— `AsyncSigmaVerify` wraps `verify()`, `challenge()`, `call()` via `asyncio.to_thread()`. No inheritance. Synchronous instantiation. Wrapping at outermost boundary covers `_call_reasoning()`/`_call_with_thinking()` which are called synchronously inside those methods.

### 4. DA Challenges That Changed the Build

**DA[#1] (HIGHEST IMPACT): KEEP_SECTIONS → REWRITE approach**
Three-agent convergence (DA + IE-BC[#4] + IE-BC[#5]) identified that the KEEP_SECTIONS allowlist retained §2h and §2a-§2e verbatim — those sections contain live tool-call instructions (`verify_finding()`, `cross_verify()`, `store_agent_memory`) that produce hallucinated output or silent skips under SDK dispatch. Forced fundamental revision of ADR[2] from allowlist to REWRITE approach. This is the load-bearing H3 mitigation.

**DA[#2]: Phase A.0 smoke gate added as hard prerequisite**
DA correctly identified that all 5 modules rest on unverified H1 (orchestrator importability). Plan restructured: SQ[TA-1a] = import integration gate (0.5d, must PASS before any other build work). Scope of smoke test upgraded from import-only to start→advance roundtrip with isolated checkpoint.

**DA[#3]: 4-item → 7-item injection**
DA evidence: sigma-comm.md provides ΣComm notation spec; without it SDK-dispatched agents produce plain English that gate_checks.py regex patterns do not recognize. Added items 5-7: sigma_comm, comms_format (Comms section from agent .md), workspace_target.

**DA[#6]: Dual stagger mechanism identified** (two-agent convergence: DA + IE-BC[#8])
Dispatch-start stagger (IC[TA-2] stagger_ms) does NOT satisfy the SC2 write-path contract. Agents starting 500ms apart can still complete near-simultaneously. Write-path serialization (`asyncio.Semaphore(1)` + sleep AFTER write) is the correct mechanism for preventing Streamlit rerun storm. Both staggers required, different purposes, different locations.

**DA[#8]: MEMORY ENTRY extraction specified for post-response persistence**
Three options assessed: structured output section (Option A), regex parsing (Option B), lossy raw text (Option C). Compromise: Option A (system= prompt instructs agent to end with `## MEMORY ENTRY` section) with Option C as degraded fallback (first 200 chars if section absent).
