# Sigma-UI Architecture Review — Synthesis Report
## Date: 2026-03-28 | Mode: ANALYZE | Team: 5+DA | Rounds: 2 | Exit-gate: PASS (A-)

### 1. Executive Summary

A five-agent plus devil's advocate team analyzed the architecture for sigma-ui, a local web application to replace chat-based sigma-review orchestration with a proper UI featuring orchestrator-as-outer-loop dispatch, belief state computation, and user gates. **The key finding, driven by DA pressure, is that the originally proposed full architecture (FastAPI + custom UI + SDK dispatch) has only a 35% probability of achieving its core goals — a below-average bet.** The recommended path, emerging from the RCA's DA-challenged reframing, is **Streamlit + Anthropic Python SDK as a scoped v1 with a graduation path** to a richer architecture if v1 succeeds. The critical DA-driven insight: the question is not "Streamlit vs FastAPI" but "Streamlit+SDK vs FastAPI+SDK" — both need the SDK dispatch layer, so framework choice is about the UI, not the orchestration.

---

### 2. Questions Answered

**Q1: Agent dispatch mechanism**
Anthropic Python SDK (`anthropic.messages.create()`) is the correct dispatch mechanism. Agent SDK v0.1.48 was initially dismissed as "0.x premature" but upgraded to "viable alternative, lower fit for phase-gate model" after DA challenge — spike test remains open (OQ-1). CLI shell-out disqualified on security and structured-return grounds. Confidence: HIGH on Python SDK; MEDIUM on Agent SDK disqualification (provisional).

**Q2: UI framework**
Streamlit with mandatory fragment-per-agent-card architecture, conditional on two burst-mitigation requirements: (1) orchestrator stagger convergence writes >=500ms, (2) `st.fragment key=agent_id`. Three agents independently converged via different methods. DA partially reopened burst-rate — if staggering infeasible, React becomes correct. Confidence: HIGH (conditional).

**Q3: Belief state computation**
Workspace observables are reliable IF parsed structurally. DA challenge produced a three-tier model: **TIER-A** (gaming-resistant — orchestrator-produced), **TIER-B** (cost-raising — structural parsing), **TIER-C** (gameable — agent-asserted). Gate checks must prioritize TIER-A. Confidence: MEDIUM-HIGH.

**Q4: Minimal viable user gate set**
3 default + 1 conditional: G1 pre-spawn confirmation (hard block), G2 DA exit-gate verdict (adaptive: modal on PASS, full-page on FAIL), G3 promotion approval (semi-auto), G4 conditional scope-drift. Cross-verified by both external models. Confidence: HIGH.

**Q5: Inter-agent communication**
Workspace-as-shared-state sufficient with private output buffers + orchestrator merge. No message bus needed. Confidence: HIGH.

**Q6: Architecture risks**
Three hard risks: tool-access loss (highest complexity + H4 risk), guard gaming residual, blocking-call event-loop stall. All agents aligned on severity.

### 4. Architecture Recommendation

**Streamlit + Anthropic Python SDK as v1, graduation path to FastAPI.**

**V1 Scope:**
- Orchestrator: `orchestrator-config.py` imported as library (85% verbatim)
- Dispatch: `anthropic.AsyncAnthropic().messages.create()` with agent .md as system prompts
- MCP: P1 orchestrator-mediated (~200 LOC context-builder, read+write atomic)
- UI: PhaseStrip + AgentGrid (fragment-per-card) + FindingsFeed + TensionPanel (round-scoped) + UserGate (3+1 adaptive)
- Belief: extend `gate_checks.py` with TIER-A observables
- Fix: async adapter class for sigma-verify sync clients
- Avoid: AsyncRunner as trust anchor

**Defer to V2+ (evidence-triggered graduation):**
- P2 tool-forwarding (trigger: quality gap from P1 limitation)
- FastAPI + SSE (trigger: Streamlit polling jank)
- Agent SDK (trigger: spike test confirms phase-gate compatibility)
- Cross-mode UI unification
- Immutable audit log

**Migration: Strangler fig, 4 phases, zero disruption to current system.**
- Phase 1 — Import orchestrator-config.py as library (coexist with current system)
- Phase 2 — FastAPI/HTTP routes wrap cmd_* logic (CLI still works)
- Phase 3 — Wire Streamlit frontend + async executor + P1 MCP integration (read+write atomic)
- Phase 4 — Decommission CLI entry points

**Component Reuse Map:**
- REUSE AS-IS: hateoas-agent Orchestrator, orchestrator-config.py, sigma-mem (MCP), sigma-verify (MCP), agent .md definitions (as system prompts)
- REUSE + EXTEND: gate_checks.py (TIER-A observable computation)
- DO NOT REUSE: AsyncRunner (elevated to HIGH risk), CLI shell-out dispatch pattern

### 7. DA Challenges That Changed the Analysis

1. **RCA DA[#3] — The 80/20 gap specification (HIGHEST IMPACT).** RCA's original "80% of goals in 20% of time" claim for Streamlit-wrapper was vague. DA forced specification of the missing 20%, which revealed that the #1 requirement (SDK-based agent dispatch to fix control inversion) was IN the missing 20%. This reframed the entire architecture question from "Streamlit vs FastAPI" to "Streamlit+SDK vs FastAPI+SDK" — framework choice is about UI, not orchestration. Load-bearing shift.

2. **RCA DA[#1] + DA[#5] — P(core goals) revised from 50% to 35%.** DA forced explicit reference-class weighting, revealing that orchestration-UI (most analogous to sigma-ui's goal) has the lowest base rate (20-30%) and should get highest weight. Separate DA challenge exposed that the 50% estimate was a judgment call, not derived from the per-hypothesis estimates. Material downward revision.

3. **TA DA[#1] — TIER-A/B/C observable model.** DA challenged "hard to fake" framing on structural observables. Tech-architect conceded and produced a three-tier model separating gaming-resistant (orchestrator-produced), cost-raising (structural-parse), and gameable (agent-asserted) observables. Structural improvement to guard design.

4. **TA DA[#3] — H2 downgraded.** Combined with DA[#1], H2 was lowered from "partially confirmed" to "weakly-to-partially confirmed." P(H2 fully works) = 55-65%, dependent on TIER-A implementation discipline.

5. **IE DA[#1] — Blocking-client reclassified from "acknowledged risk" to "design defect."** Convention-only enforcement of async discipline + silent total stall on violation = design defect requiring architectural fix (async adapter class), not just documentation.

6. **UXR DA[#1] — G2 adaptive tiering.** G2 gate changed from blanket full-page to TIER-2 modal on PASS, TIER-1 full-page on FAIL. Aligns gate weight with action reversibility.

7. **RCA DA[#4] — P=52% quality argument strengthened Streamlit+SDK case.** At coin-flip odds on quality preservation, the simpler architecture (fewer integration points for quality leaks) has higher P(H4). This became an additional argument for Streamlit+SDK v1 over the full architecture.
