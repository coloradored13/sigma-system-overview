PROMOTE[CQA-P1][user-approve][anti-pattern-new]: deferred-scope-trap
When a feature's scope is reduced to "types only" (lifecycle deferred to a later phase), the consuming code that depends on that feature (e.g. a hardcoded None sentinel in a snapshot method) is left in a state that produces silent wrong data when the feature is eventually added. The trap: the hardcoded sentinel is syntactically valid and passes all current tests, but the future implementer must remember to update BOTH the new method AND the consumer code.
