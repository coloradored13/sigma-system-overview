#### ADR[B2-3]: Gate Lifecycle Design — 5 Decisions (responding to IE + CQA challenges)

(1) active_gate field: GateRequestDict | None on _ReviewSession (single gate, ¬stack). Phase B2 gates are sequential.
(2) fire_gate() signature: fire_gate(gate: GateRequestDict) → None. Alias for set_active_gate(gate). Caller constructs GateRequestDict with status="FIRING".
(3) resolve_gate() behavior: sets active_gate=None (cleared). ¬mutates in-place. Returns GateActionDict.
(4) gate_action_log: EVENTS-ONLY. Gate events in self.events via _emit_event. No separate list. No Phase B2 UI consumer for gate history. Consistent with event-append pattern.
(5) gate methods are SYNC (¬async). Not concurrent with convergence. _write_lock NOT needed (user-driven, sequential). IE recommendation confirmed.

DOUBLE-FIRE POLICY (CQA challenge): raise GateConflictError if fire_gate() called while active_gate is not None. Consistent with KeyError-on-unknown-id pattern in add_tension_response(). Exception declared in review_state.py at module level. ¬silently replace (would mask orchestrator bugs). ¬silently ignore (would lose gate events).

METHODS:
  _ReviewSession.set_active_gate(gate: GateRequestDict | None) → None
    if gate is not None and self.active_gate is not None: raise GateConflictError
    self.active_gate = gate
    event: "gate_fired" or "gate_cleared"
  _ReviewSession.resolve_gate(gate_id: str, outcome: str) → GateActionDict
    validate outcome in {"PASSED","FAILED"}
    validate self.active_gate is not None and self.active_gate["id"] == gate_id
    self.active_gate = None
    event: "gate_resolved"
    return GateActionDict(gate_id=gate_id, action="resolve", outcome=outcome, timestamp=time.time())
  ReviewState.set_active_gate(review_id, gate) → delegate
  ReviewState.resolve_gate(review_id, gate_id, outcome) → GateActionDict delegate
