Three poll calls required per rerun: get_state() + get_findings(round_num=N) + get_tensions(round_num=N). Methods exist on ReviewState (lines 601-632). Sources: code-read review_state.py (T1).

**H[4] CONFIRMED**: st.dialog is stable from v1.37.0 (July 25 2024 GA). Single-dialog constraint applies: only one dialog per script run. G2 is the only TIER-2 gate so no conflict in normal flow. dismissible=False required for G1; G2 dismiss = cancel (not auto-pass). Sources: docs.streamlit.io release notes 2024 (T1, independently verified during DA[#7]).

## Architecture Decisions (locked)

**ADR[B2-1]: OrchestratorWrapper Session Persistence**
Decision: Store OrchestratorWrapper as Python object ref in ss["orch_wrapper"]. Auto-checkpoint after every advance() to fixed path (.sigma_ui_checkpoint.json). On ss init, restore from checkpoint if file exists (recovery path). enforceSerializableSessionState=False (default) — document in .env.example as incompatible if set.
Rationale: session_state heap objects survive reruns; no serialization needed. Checkpoint-on-advance provides durability at near-zero per-rerun cost.
Alternatives rejected: checkpoint roundtrip per rerun (100ms overhead + JSON loses live module refs); rebuild per cold-start (cold start = session destroyed = review must restart).

**ADR[B2-2]: Fragment Strategy — Single AgentGrid Fragment (Lead Decision)**
Decision: Single @st.fragment(run_every=2.0) wrapping entire AgentGrid. Default-arg capture on _rs and _rid. Agents sorted by agent_id for stable call order. Architecture supports upgrade to per-agent fragments if profiling shows need.
Rationale: Per-agent approach (20 concurrent timers) was T3 unverified (XVERIFY-REQUESTED). Single fragment meets UX goals (cards update every 2s) and satisfies C[3] in spirit. Per-agent fragments are a Phase B3 optimization.
Alternatives rejected: 20 per-agent @st.fragment(run_every) timers (unverified at scale, premature optimization).

**ADR[B2-3]: Gate Lifecycle — GateConflictError on Double-Fire, SYNC Methods, Events-Only Log**
Decision: GateConflictError declared at module level in review_state.py (not types.py). fire_gate() raises GateConflictError if active_gate already set (not silent replace). resolve_gate() sets active_gate=None, returns GateActionDict, does not mutate in-place. Gate methods are SYNC. Events-only audit via _emit_event (no separate gate_action_log).
Rationale: GateConflictError consistent with KeyError-on-unknown-id pattern in add_tension_response(). Silent replace masks orchestrator bugs. Sync methods avoid _write_lock contention with convergence stagger.
Alternatives rejected: silent replace (masks double-fire bugs); silent ignore (loses gate events); async methods (would require _write_lock held during gate writes).

**ADR[B2-4]: PhaseStateDict.active_gate Type Fix**
Decision: Change types.py PhaseStateDict.active_gate from `dict | None` to `GateRequestDict | None`.
Rationale: Only bare `dict` field in entire TypedDict surface. Enables mypy --strict clean output and precise test assertions on gate shape.
Alternatives rejected: keep bare dict (produces spurious Any warnings; test assertions against gate fields not statically validated).

**ADR[B2-5]: active_gate Hardcode Fix**
Decision: review_state.py:347 — active_gate=None → active_gate=self.active_gate. _ReviewSession.__init__ adds active_gate: GateRequestDict | None = None field.
Rationale: Prerequisite for IC5 gate lifecycle; without this fix the gate UI never fires.

**ADR[B2-6]: Component Data-Prep/Render Separation**
Decision: Each component module has a pure-function data-prep layer (no st.* imports, required tests) and a render layer (st.* calls, no test required). Canonical function names:
- build_phase_node_display(node: PhaseNodeStateDict) → PhaseNodeDisplay
- build_agent_card_display(agent: AgentStateDict) → AgentCardDisplay
- build_finding_row_display(finding: FindingDict) → FindingRowDisplay
- build_tension_thread_display(tension: TensionItemDict) → TensionThreadDisplay
- route_gate_tier(gate: GateRequestDict | None) → Literal["TIER-1","TIER-2","TIER-3","NONE"]
Return types are NamedTuples/TypedDicts (not bare dicts). >= 1 test per function required.
Rationale: Preserves existing zero-Streamlit-import test pattern. Data transformation logic in render functions would be untestable.
Alternatives rejected: logic embedded in render/fragment bodies (untestable, violates established testing pattern).

**ADR[B2-7]: Component File Organization**
Decision: sigma_ui/components/__init__.py (exports), phase_strip.py, agent_grid.py, findings_feed.py, tension_panel.py, user_gate.py; app.py at repo root.
Rationale: Clean separation; each component is independently importable and testable.
Note on TensionPanel: decisions.md L5 specified "drawer" but Streamlit has no native drawer. st.expander confirmed as acceptable substitute by product-designer (OQ-B2-5). Achieves identical deliberate-friction UX intent. Custom HTML/CSS rejected (maintenance burden, breaks theme).

**ADR[B2-8]: Streamlit Version Constraint**
Decision: streamlit>=1.37.0 in pyproject.toml [project].dependencies. (Build landed at >=1.40.0 — conservative drift, harmless.)
Rationale: Both st.fragment (with run_every) and st.dialog graduated to stable in v1.37.0 (July 25 2024). Earlier TA claim of v1.36.0 and PD claim of v1.55.0 were both wrong; corrected via T1-verified release notes (DA[#7] resolution).
Alternatives rejected: >=1.36.0 (st.dialog still experimental); >=1.55.0 (unnecessarily restrictive).

**ADR[B2-9]: 8 Skipped DS Enum Tests — Deferred to Phase B3**
Decision: DEFER. Scope-boundary updated with explicit "Phase B3 enum canonicalization" label.
Rationale: Tests require AgentStatus, GateStatus, GateTier, GateActionRole, XVerifyStatus, SourceTier, ObservableTier enums — none exist in types.py. Adding them is non-trivial and affects dispatcher, review_state, tier_a_observables. DispatchMode.CLI vs LOCAL naming drift also deferred.

**ADR[B2-10]: Preflight Architecture — sigma_ui/preflight.py Standalone Module**
Decision: Pre-flight as sigma_ui/preflight.py callable from both setup.sh and app.py startup. @dataclass(frozen=True) PreflightResult(ok: bool, messages: tuple[str, ...]). run_preflight(config_path: Path | None, gate_path: Path | None) → PreflightResult.
Rationale: Avoids duplicating check logic across setup-time and runtime entry points.
Note: Build landed with mutable @dataclass (DA-BUILD[#2]); frozen=True is the locked spec. Also, build used str | None for config_path (DA-BUILD[#3] accepted as more ergonomic at call sites).

## Design System

### DS Tokens (Streamlit-native, no custom CSS)

- DS[status-color]: `:color[symbol+label]` syntax — ACTIVE=:green[● ACTIVE] | CONVERGED=:blue[✓ CONVERGED] | BLOCKED=:red[⊘ BLOCKED] | ERROR=:red[⚠ ERROR] | STALE=:orange[○ STALE] | IDLE=:gray[○ IDLE]
- DS[source-badges]: independent-research=:green | prompt-claim=:orange | cross-agent=:blue | agent-inference=:gray | external-verification=:green
- DS[xverify-badges]: CONFIRMED=:green[XVERIFY✓] | PARTIAL=:orange[XVERIFY~] | FAILED=:red[XVERIFY✗] | NONE=(omit)
- DS[da-badge]: is_da_finding=True → :red[**DA**] inline in FindingCard header row
- DS[verdict-badges]: CONCEDE=:green[CONCEDE] | COMPROMISE=:orange[COMPROMISE] | DEFEND=:blue[DEFEND] | PENDING=:gray[PENDING]
- DS[gate-badges]: TIER-1=:red[TIER-1 GATE] | TIER-2=:orange[TIER-2 GATE] | TIER-3=:blue[TIER-3 NOTICE]
- DS[accessibility]: informational only (not a build requirement). Color + symbol for status (not color-alone).
- DS[responsive]: informational only (not a build requirement). Min-usable 1280px, mobile out of scope.

### IX Patterns

- Outer layout: st.columns([3,1]) main-content:meta-sidebar
- Phase strip: st.columns(n_phases+1, gap="small") — trailing col for belief metric
- Agent grid: st.columns(4, gap="small") with row-wrapping; st.container(border=True) per card
- Cards: st.container(border=True) for AgentCard, FindingCard, TensionThread, GateCard
- Drill-down: st.expander for AgentDetail, FindingFull, TensionPanel-entry, TensionResponse
- KPI: st.metric for belief-score and round-counter
- Convergence: st.progress(value=resolved/total, text=label) in TensionPanel
- Empty states: st.info() for all empty lists; st.spinner for loading; st.warning for TIER-3 notices
- Typography: st.markdown("## Section") for headers; inline color via :color[] syntax

## Interface Contracts

**IC[B2-1]: app.py Session State Machine**

Persisted-once keys (init guard — if "orch_wrapper" not in ss):
- ss["orch_wrapper"]: OrchestratorWrapper
- ss["review_state"]: ReviewState
- ss["review_id"]: str
- ss["review_started"]: bool = False
- ss["g1_passed"]: bool = False
- ss["g4_ack"]: bool = False
- ss["findings_page"]: int = 0
- ss["tensions_page"]: int = 0
- ss["findings_round_filter"]: int | None = None
- ss["advance_in_progress"]: bool = False

Transient keys (set and cleared within same flow):
- ss["gate_passed"]: bool
- ss["gate_outcome"]: str | None

Computed-per-rerun (not stored in ss):
- snapshot — from get_state(review_id)
- findings — from get_findings(review_id, round_num, page)
- tensions — from get_tensions(review_id, round_num, page)
- active_gate — from snapshot["phase"]["active_gate"]

Rendering order (gate checks before dashboard):
1. Preflight check (if not ok → st.error + st.stop())
2. Init guard (if "orch_wrapper" not in ss → create all keys + optional checkpoint restore)
3. Start guard (if not ss["review_started"] → render start UI + st.stop() or call start())
4. 3-call poll: get_state() + get_findings() + get_tensions()
5. TIER-1 gate check (if active_gate and tier=="TIER-1" → render_gate() + st.stop())
6. Advance trigger (if ss["gate_passed"] → advance_and_clear())
7. TIER-2 gate (if active_gate and tier=="TIER-2" → call @st.dialog fn)
8. TIER-3 gate (if active_gate and tier=="TIER-3" → st.warning inline)
9. Dashboard: render_phase_strip() | render_agent_grid() | render_findings_feed() | render_tension_panel()
10. Meta-sidebar: st.metric(round) + st.metric(agents) + advance-button (if not terminal) + caption

**IC[B2-2]: Component Render Signatures (canonical — DA[#1] resolution)**

Data-prep layer (pure Python, no st.* imports, required tests):
- phase_strip.py: build_phase_node_display(node: PhaseNodeStateDict) → PhaseNodeDisplay
- agent_grid.py: build_agent_card_display(agent: AgentStateDict) → AgentCardDisplay
- findings_feed.py: build_finding_row_display(finding: FindingDict) → FindingRowDisplay
- tension_panel.py: build_tension_thread_display(tension: TensionItemDict) → TensionThreadDisplay
- user_gate.py: route_gate_tier(gate: GateRequestDict | None) → Literal["TIER-1","TIER-2","TIER-3","NONE"]

Render layer (calls st.* — no test required):
- render_phase_strip(state: ReviewState, review_id: str) → None
- render_agent_grid(state: ReviewState, review_id: str) → None
- render_findings_feed(state: ReviewState, review_id: str) → None
- render_tension_panel(state: ReviewState, review_id: str) → None
- render_gate(tier: str, gate: GateRequestDict, state: ReviewState, review_id: str) → None

**IC[B2-3]: Gate Lifecycle Call Sequence**

G1 (TIER-1): orchestrator fires → state.fire_gate(review_id, GateRequestDict(id="G1",tier="TIER-1",...)) → snapshot.phase.active_gate non-None → render_gate("TIER-1",...) → st.stop() → user confirms → resolve_gate(review_id,"G1","PASSED") → ss["gate_passed"]=True + ss["g1_passed"]=True → st.rerun() → advance_and_clear()

G2 (TIER-2): state.fire_gate(review_id, GateRequestDict(id="G2",tier="TIER-2",...)) → render dashboard → @st.dialog(dismissible=True) opens → PASS/FAIL → resolve + ss["gate_passed"]=True → st.rerun() inside dialog → advance_and_clear(). DISMISS → gate remains open, dialog reopens next rerun.

G4 (TIER-3): state.fire_gate(review_id, GateRequestDict(id="G4",tier="TIER-3",...)) → st.warning banner + Acknowledge button → Acknowledge → resolve + ss["g4_ack"]=True + ss["gate_passed"]=True → st.rerun()

**IC[B2-4]: Gate Method Signatures on ReviewState (public API)**

- fire_gate(review_id: str, gate: GateRequestDict) → None — alias that pre-sets status="FIRING", calls set_active_gate. Raises GateConflictError if gate already active.
- resolve_gate(review_id: str, gate_id: str, outcome: str) → GateActionDict — validates gate_id match, sets active_gate=None, returns GateActionDict.
- GateConflictError: module-level exception in review_state.py.
- Note: IC specified set_active_gate as public, but build omitted it (DA-BUILD[#1] accepted — fire_gate is the better abstraction).

**IC[B2-5]: Preflight Interface**

- @dataclass(frozen=True) PreflightResult(ok: bool, messages: tuple[str, ...])
- run_preflight(config_path: Path | None = None, gate_path: Path | None = None) → PreflightResult
- Checks: ANTHROPIC_API_KEY in env, streamlit importable, hateoas_agent importable, sigma_mem importable, config_path/gate_path existence (if provided)
