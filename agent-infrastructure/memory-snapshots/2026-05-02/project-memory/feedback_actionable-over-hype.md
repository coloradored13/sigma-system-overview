When synthesizing sigma-review findings, lead toward IMPLEMENTATION not POSITIONING. The user flagged the Kaggle AGI benchmark synthesis as "hyping up a win" — too much strategy-speak ("novel differentiator," "what makes this win," "gatekeeper/differentiator framework") and not enough concrete buildable detail.

**Why:** The user is a builder, not a strategy consumer. They need to know what the code looks like, what the actual test items are, what's genuinely novel vs framing. "Zero prior benchmarks exist" is a claim — show WHY by explaining what the test actually does differently.

**How to apply:** In synthesis, lead with implementation detail (code patterns, item examples, scoring mechanics) over strategic framing. Strategy context is fine but secondary. When agents produce concrete designs (TA's SDK code, CDS's paradigm structures), preserve that detail in synthesis rather than abstracting it away. The user can derive strategy from concrete plans; they can't derive concrete plans from strategy.
