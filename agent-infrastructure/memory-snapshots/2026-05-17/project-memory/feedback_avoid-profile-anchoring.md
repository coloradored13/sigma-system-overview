When the user explicitly asks for an evidence-based opportunity landscape, market analysis, or "what are the options" question, do NOT anchor the synthesis on the user's profile, skills, domain expertise, or "edges." Personalizing the answer narrows scope, biases recommendations toward what the user can easily build, and loses value relative to what the user actually asked for.

**Why:** User explicitly corrected during /sigma-single on K-shape business opportunities (26.5.17): "this shouldn't anchor on me and my abilities at all. Doesn't have to focus on AI, Financial Services, Product, etc. Airplane painting could be the play for example, or executive dog walker, or AI data center security guard. I'm more interested in the evidence, not how the evidence relates to me." My first synthesis ranked "loan-agency operational tooling" as #1 because it matched user's network/domain — that was overfitting to user identity, not following evidence.

**How to apply:**
- When task is landscape mapping or evidence synthesis, deliver the evidence as-is. Include weird, low-tech, blue-collar, service-economy, physical-labor, skilled-trade, SMB-acquisition, and franchising opportunities alongside obvious tech/software plays.
- Default opportunity types to cover: tech/SaaS, services-for-affluent (concierge medicine, executive dog walking, longevity clinics, MedSpa), physical-labor adjacent to gold rushes (data center construction, transmission line workers, nuclear welders), skilled trades (HVAC, plumbing, specialized painting), SMB roll-up/acquisition (silver tsunami plays), trade-down enablers (resale, repair, value retail), healthcare home services.
- Only personalize ("your edge", "your plays", "for your profile") when the user explicitly invites that framing.
- User's profile (memory U[] entry) is for tailoring communication style and tone, NOT for narrowing analytical scope.

This is a counterweight to the natural "please tendency" flagged in sigma-mem core — anchoring on user's profile feels helpful but is actually a narrowing bias when the user wants neutral evidence. Related: [[feedback_no-bias-no-pleasing]].
